## 1. VISIÓN Y CONCEPTO

### El Problema Actual
- Navegación manual entre /analytics y /automation
- No hay contexto compartido entre dashboards
- UI clásica requiere buscar opciones en menús
- Sin capacidad de procesamiento conversacional de datos

### La Solución: Dashboard Orquestado por IA

```
┌────────────────────────────────────────────────────────────────┐
│                      RANDOM COMMAND CENTER                      │
├──────────────┬─────────────────────────────────────────────────┤
│   SIDEBAR    │              WORKSPACE                          │
│   (Tabs)     │                                                 │
│              │  ┌──────────────────────────────────────┐       │
│  🔬 Labs     │  │                                      │       │
│  📊 Analytics│  │         CONTENIDO DINÁMICO           │       │
│  🤖 Automation│ │      (Widgets, Charts, Tables)       │       │
│              │  │                                      │       │
│  + Futuro    │  └──────────────────────────────────────┘       │
│              │                                                 │
├──────────────┴─────────────────────────────────────────────────┤
│                    AI COPILOT PANEL                            │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │ 💬 "¿Cuántos leads tengo en total?"                       │ │
│  │ 🤖 "Tienes 47 leads. 12 pendientes, 28 aprobados..."     │ │
│  │    [Widget: Lead Distribution Chart appears above ↑]      │ │
│  └──────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────┘
```

### Principios de Diseño

1. **Conversación > Navegación**
   - El copiloto entiende intenciones, no comandos específicos
   - Responde con widgets dinámicos, no solo texto

2. **Contexto Compartido**
   - Conoce qué tab estás viendo
   - Acceso a datos de Analytics, Automation y Labs
   - Memoria de conversación durante la sesión

3. **Orquestación de UI**
   - Puede modificar filtros en tiempo real
   - Genera y actualiza widgets bajo demanda
   - Navega automáticamente entre secciones

### Copilot Panel Component

```jsx
// CopilotPanel.jsx
import { useState, useRef, useEffect } from 'react';
import { useMutation } from 'react-query';
import ReactMarkdown from 'react-markdown';

const CopilotPanel = ({ context, onWidgetGenerate, onContextUpdate }) => {
  const [messages, setMessages] = useState([
    {
      role: 'assistant',
      content: '¡Hola! Soy tu copiloto. ¿En qué puedo ayudarte?',
    }
  ]);
  const [input, setInput] = useState('');
  const messagesEndRef = useRef(null);

  const { mutate: sendMessage, isLoading } = useMutation(
    async (userMessage) => {
      const response = await fetch('/api/copilot/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          message: userMessage,
          context: context,
          history: messages,
        }),
      });
      return response.json();
    },
    {
      onSuccess: (data) => {
        // Add AI response to messages
        setMessages(prev => [...prev, {
          role: 'assistant',
          content: data.text,
        }]);

        // Generate widgets if provided
        if (data.widgets && data.widgets.length > 0) {
          data.widgets.forEach(widget => onWidgetGenerate(widget));
        }

        // Update context if AI changed it
        if (data.context_update) {
          onContextUpdate(prev => ({ ...prev, ...data.context_update }));
        }
      }
    }
  );

  const handleSubmit = (e) => {
    e.preventDefault();
    if (!input.trim()) return;

    // Add user message
    const userMessage = input.trim();
    setMessages(prev => [...prev, {
      role: 'user',
      content: userMessage,
    }]);
    setInput('');

    // Send to AI
    sendMessage(userMessage);
  };

  useEffect(() => {
    messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [messages]);

  return (
    <div className="copilot-panel">
      <div className="copilot-header">
        <h3>🤖 AI Copilot</h3>
        <span className="context-indicator">
          {context.currentTab}
        </span>
      </div>

      <div className="messages-container">
        {messages.map((msg, idx) => (
          <div key={idx} className={`message ${msg.role}`}>
            <div className="message-avatar">
              {msg.role === 'user' ? '👤' : '🤖'}
            </div>
            <div className="message-content">
              <ReactMarkdown>{msg.content}</ReactMarkdown>
            </div>
          </div>
        ))}
        {isLoading && (
          <div className="message assistant">
            <div className="message-avatar">🤖</div>
            <div className="message-content typing">
              <span></span><span></span><span></span>
            </div>
          </div>
        )}
        <div ref={messagesEndRef} />
      </div>

      <form className="input-form" onSubmit={handleSubmit}>
        <input
          type="text"
          value={input}
          onChange={(e) => setInput(e.target.value)}
          placeholder="Pregunta algo sobre tus datos..."
          disabled={isLoading}
        />
        <button type="submit" disabled={isLoading || !input.trim()}>
          ➤
        </button>
      </form>
    </div>
  );
};
```

```jsx
                nameKey="label"

              />
            </PieChart>
          </div>
        );

      case 'bar_chart':
        return (
          <div className="widget-card">
            <h4>{widget.title}</h4>
            <BarChart width={400} height={300} data={widget.data}>
              <Bar dataKey="value" fill="#82ca9d" />
            </BarChart>
          </div>
        );

      case 'metric_card':
        return (
          <div className="widget-card metric">
            <h4>{widget.title}</h4>
            <div className="metric-value">{widget.value}</div>
            <div className="metric-change">{widget.change}</div>
          </div>
        );

      case 'data_table':
        return (
          <div className="widget-card">
            <h4>{widget.title}</h4>
            <table>
              <thead>
                <tr>
                  {widget.columns.map(col => <th key={col}>{col}</th>)}
                </tr>
              </thead>
              <tbody>
                {widget.rows.map((row, idx) => (
                  <tr key={idx}>
                    {row.map((cell, i) => <td key={i}>{cell}</td>)}
                  </tr>
                ))}
              </tbody>
            </table>
          </div>
        );

      default:
        return null;
    }
  };

  return renderWidget();
};
```

## 5. SISTEMA DE IA - ESTRATEGIA HÍBRIDA

### Problema: Balance entre Costo y Calidad

**Objetivo**: Maximizar la calidad de respuestas mientras minimizamos costos operativos.

### Estrategia: LLM Router Inteligente

```python
# backend/ai/llm_router.py

from enum import Enum
from typing import Optional
import tiktoken

class QueryComplexity(Enum):
    SIMPLE = "simple"          # Queries de 1 tool call, respuesta directa
    MODERATE = "moderate"      # Múltiples tools, agregación básica
    COMPLEX = "complex"        # Razonamiento multi-step, análisis profundo

class LLMRouter:
    """
    Decide qué modelo usar basándose en:
    - Complejidad de la query
    - Presupuesto disponible del usuario
    - Rate limits
    """
    
    def __init__(self):
        self.free_models = {
            "gemini-flash-1.5-exp": {
                "provider": "openrouter",
                "rpm": 60,  # requests per minute
                "tpm": 4_000_000,  # tokens per minute
                "cost_per_1m_input": 0.0,
                "cost_per_1m_output": 0.0,
                "best_for": [QueryComplexity.SIMPLE],
            },
            "llama-3.1-70b-instruct": {
                "provider": "groq",
                "rpm": 30,
                "tpm": 6_000,
                "cost_per_1m_input": 0.0,
                "cost_per_1m_output": 0.0,
                "best_for": [QueryComplexity.SIMPLE, QueryComplexity.MODERATE],
            },
        }
        
        self.premium_models = {
            "claude-3.5-sonnet": {
                "provider": "anthropic",
                "rpm": 50,
                "tpm": 40_000,
                "cost_per_1m_input": 3.0,
                "cost_per_1m_output": 15.0,
                "best_for": [QueryComplexity.COMPLEX],
            },
            "gpt-4-turbo": {
                "provider": "openai",
                "rpm": 500,
                "tpm": 150_000,
                "cost_per_1m_input": 10.0,
                "cost_per_1m_output": 30.0,
                "best_for": [QueryComplexity.COMPLEX],
            },
        }
    
    def classify_query_complexity(
        self, 
        message: str, 
        context: dict
    ) -> QueryComplexity:
        """
        Clasifica la complejidad de una query basándose en:
        - Keywords
        - Longitud del mensaje
        - Número de datos requeridos
        - Si requiere múltiples pasos de razonamiento
        """
        # Keywords que indican queries simples
        simple_keywords = [
            "cuántos", "cuántas", "total", "sum", "count",
            "muéstrame", "dame", "lista", "ver"
        ]
        
        # Keywords que indican queries complejas
        complex_keywords = [
            "analiza", "compara", "recomienda", "predice",
            "optimiza", "estrategia", "insights", "tendencias",
            "correlación", "impacto", "por qué"
        ]
        
        message_lower = message.lower()
        
        # Check for complex patterns first
        if any(kw in message_lower for kw in complex_keywords):
            return QueryComplexity.COMPLEX
        
        # Check for simple patterns
        if any(kw in message_lower for kw in simple_keywords):
            return QueryComplexity.SIMPLE
        
        # Default to moderate
        return QueryComplexity.MODERATE
    
    def select_model(
        self,
        complexity: QueryComplexity,
        user_tier: str = "free",  # "free" or "premium"
        estimated_tokens: int = 1000,
    ) -> dict:
        """
        Selecciona el mejor modelo basándose en:
        - Complejidad
        - Tier del usuario
        - Tokens estimados
        """
        if user_tier == "premium":
            # Usuario premium puede usar cualquier modelo
            if complexity == QueryComplexity.COMPLEX:
                return {
                    "model": "claude-3.5-sonnet",
                    **self.premium_models["claude-3.5-sonnet"]
                }
            elif complexity == QueryComplexity.MODERATE:
                return {
                    "model": "llama-3.1-70b-instruct",
                    **self.free_models["llama-3.1-70b-instruct"]
                }
            else:
                return {
                    "model": "gemini-flash-1.5-exp",
                    **self.free_models["gemini-flash-1.5-exp"]
                }
        
        else:  # user_tier == "free"
            # Usuario free solo usa modelos gratuitos
            if complexity == QueryComplexity.COMPLEX:
                # Para queries complejas, sugerir upgrade pero usar el mejor free
                return {
                    "model": "llama-3.1-70b-instruct",
                    "suggest_upgrade": True,
                    **self.free_models["llama-3.1-70b-instruct"]
                }
            else:
                return {
                    "model": "gemini-flash-1.5-exp",
                    **self.free_models["gemini-flash-1.5-exp"]
                }
    
    async def route_query(
        self,
        message: str,
        context: dict,
        user_tier: str = "free"
    ) -> dict:
        """
        Pipeline completo:
        1. Clasificar complejidad
        2. Estimar tokens
        3. Seleccionar modelo
        4. Retornar configuración
        """
        complexity = self.classify_query_complexity(message, context)
        
        # Estimar tokens usando tiktoken
        enc = tiktoken.get_encoding("cl100k_base")
        estimated_tokens = len(enc.encode(message)) + 500  # +500 for context
        
        model_config = self.select_model(complexity, user_tier, estimated_tokens)
        
        return {
            "complexity": complexity.value,
            "model_config": model_config,
            "estimated_tokens": estimated_tokens,
        }
```

```python
            """

            params.append(filters["min_score"])
        
        # ... más filtros

    async def process_message(
        self,
        message: str,
        context: Dict,
        user_tier: str = "free",
        history: List[Dict] = None
    ) -> Dict:
        """
        Procesa un mensaje del usuario:
        1. Ruta el mensaje al modelo apropiado
        2. Ejecuta tools si es necesario
        3. Genera widgets si aplica
        4. Retorna respuesta estructurada
        """

        routing_info = await self.router.route_query(message, context, user_tier)
        model_config = routing_info["model_config"]
        
        # 2. Inicializar LLM según provider
        if model_config["provider"] == "anthropic":
            llm = ChatAnthropic(
                model=model_config["model"],
                temperature=0.7,
                anthropic_api_key=os.getenv("ANTHROPIC_API_KEY")
            )
        elif model_config["provider"] == "openrouter":
            llm = ChatOpenAI(
                model=model_config["model"],
                openai_api_base="https://openrouter.ai/api/v1",
                openai_api_key=os.getenv("OPENROUTER_API_KEY"),
                temperature=0.7,
            )
        elif model_config["provider"] == "groq":
            llm = ChatOpenAI(
                model=model_config["model"],
                openai_api_base="https://api.groq.com/openai/v1",
                openai_api_key=os.getenv("GROQ_API_KEY"),
                temperature=0.7,
            )
        
        # 3. Crear prompt con contexto
        prompt = ChatPromptTemplate.from_messages([
            ("system", self._get_system_prompt(context)),
            MessagesPlaceholder(variable_name="chat_history"),
            ("human", "{input}"),
            MessagesPlaceholder(variable_name="agent_scratchpad"),
        ])
        
        # 4. Crear agente con tools
        agent = create_openai_tools_agent(llm, self.tools, prompt)
        agent_executor = AgentExecutor(agent=agent, tools=self.tools, verbose=True)
        
        # 5. Ejecutar agente
        result = await agent_executor.ainvoke({
            "input": message,
            "chat_history": history or [],
        })
        
        # 6. Procesar resultado y extraer widgets
        response = self._parse_agent_output(result)
        
        # 7. Agregar metadata
        response["model_used"] = model_config["model"]
        response["complexity"] = routing_info["complexity"]
        
        if model_config.get("suggest_upgrade"):
            response["upgrade_suggestion"] = (
                "💡 Esta query podría beneficiarse de un modelo premium. "
                "Considera upgrade para análisis más profundos."
            )
        
        return response
    
    def _get_system_prompt(self, context: Dict) -> str:
        """
        Genera el system prompt dinámicamente según el contexto.
        """
        current_tab = context.get("current_tab", "analytics")

4. Si no tienes certeza, di "déjame verificar..." y usa un tool
5. Mantén el tono casual pero profesional (estilo Random)

"""
        context_info = f"""

- Filtros aplicados: {context.get('active_filters', {})}
- Widgets visibles: {len(context.get('visible_widgets', []))}

        return base_prompt + context_info
    
    def _parse_agent_output(self, agent_result: Dict) -> Dict:
        """
        Parsea el output del agente y lo estructura para el frontend.
        """
        output_text = agent_result.get("output", "")
        
        # Detectar si se generaron widgets (basándose en tool calls)
        widgets = []
        intermediate_steps = agent_result.get("intermediate_steps", [])
        
        for step in intermediate_steps:
            tool_name = step[0].tool
            tool_input = step[0].tool_input
            tool_output = step[1]
            
            if tool_name == "generate_widget":
                widgets.append(tool_output)
        
        return {
            "text": output_text,
            "widgets": widgets,
            "context_update": {},  # Para actualizar filtros/navegación
            "actions": [],  # Botones de acción sugeridos
        }
```

### 3. Strategy Pattern (LLM Selection)

```python
# backend/ai/strategies.py

from abc import ABC, abstractmethod

class LLMStrategy(ABC):
    """
    Interface para diferentes estrategias de LLM.
    Permite swapping fácil entre providers.
    """
    
    @abstractmethod
    async def generate(self, prompt: str, **kwargs) -> str:
        pass
    
    @abstractmethod
    def count_tokens(self, text: str) -> int:
        pass

class OpenRouterStrategy(LLMStrategy):
    """Implementación para OpenRouter (modelos free)"""
    
    def __init__(self, api_key: str, model: str):
        self.api_key = api_key
        self.model = model
        self.client = ChatOpenAI(
            model=model,
            openai_api_base="https://openrouter.ai/api/v1",
            openai_api_key=api_key,
        )
    
    async def generate(self, prompt: str, **kwargs) -> str:
        response = await self.client.agenerate([prompt])
        return response.generations[0][0].text
    
    def count_tokens(self, text: str) -> int:
```

```python
# backend/ai/commands.py

from abc import ABC, abstractmethod

class Command(ABC):
    """
    Interface para comandos ejecutables.
    Permite deshacer acciones (undo/redo).
    """
    
    @abstractmethod
    async def execute(self) -> Any:
        pass
    
    @abstractmethod
    async def undo(self) -> Any:
        pass

class FilterLeadsCommand(Command):
    """Comando para filtrar leads"""
    
    def __init__(self, repository, filters: Dict):
        self.repo = repository
        self.filters = filters
        self.previous_results = None
    
    async def execute(self) -> List[Lead]:
        results = await self.repo.filter(**self.filters)
        self.previous_results = results
        return results
    
    async def undo(self) -> List[Lead]:
        # Retornar a estado anterior (sin filtros)
        return await self.repo.get_all()

class ApproveLeadCommand(Command):
    """Comando para aprobar un lead"""
    
    def __init__(self, repository, lead_id: int):
        self.repo = repository
        self.lead_id = lead_id
        self.previous_status = None
    
    async def execute(self) -> Lead:
        lead = await self.repo.get_by_id(self.lead_id)
        self.previous_status = lead.status
        
        await self.repo.update_status(self.lead_id, "approved")
        return await self.repo.get_by_id(self.lead_id)
    
    async def undo(self) -> Lead:
        await self.repo.update_status(self.lead_id, self.previous_status)
        return await self.repo.get_by_id(self.lead_id)
```

### Caching Strategy

```python
# backend/ai/cache_manager.py

import hashlib
import json
from typing import Optional, Any
from redis import Redis

class CacheManager:
    """
    Gestiona caching de queries y responses para minimizar llamadas a LLM.
    """
    
    def __init__(self, redis_client: Redis):
        self.redis = redis_client
        self.ttl_short = 300  # 5 min
        self.ttl_medium = 3600  # 1 hour
        self.ttl_long = 86400  # 24 hours
    
    def _generate_key(self, message: str, context: Dict) -> str:
        """
        Genera cache key basándose en mensaje + contexto.
        Queries similares comparten cache.
        """
        # Normalize message (lowercase, strip whitespace)
        normalized = message.lower().strip()
        
        # Extract relevant context (ignore session_id, timestamps)
        relevant_context = {
            "tab": context.get("current_tab"),
            "filters": context.get("active_filters", {}),
        }
        
        # Create hash
        key_string = f"{normalized}:{json.dumps(relevant_context, sort_keys=True)}"
        hash_key = hashlib.sha256(key_string.encode()).hexdigest()
        
        return f"copilot:cache:{hash_key}"
    
    async def get(self, message: str, context: Dict) -> Optional[Dict]:
        """Obtiene response cacheada si existe"""
        key = self._generate_key(message, context)
        cached = await self.redis.get(key)
        
        if cached:
            return json.loads(cached)
        return None
    
    async def set(
        self, 
        message: str, 
        context: Dict, 
        response: Dict,
        ttl: Optional[int] = None
    ):
        """Guarda response en cache"""
        key = self._generate_key(message, context)
        
        # Determine TTL based on query type
        if not ttl:
            # Data-heavy queries: short TTL (datos cambian frecuentemente)
            if any(kw in message.lower() for kw in ["cuántos", "total", "count"]):
                ttl = self.ttl_short
            # Insights/analysis: medium TTL
            elif any(kw in message.lower() for kw in ["analiza", "insights", "tendencias"]):
                ttl = self.ttl_medium
            # General queries: short TTL
            else:
                ttl = self.ttl_short
        
        await self.redis.setex(
            key,
            ttl,
            json.dumps(response)
        )
```

### Rate Limiting

```python
# backend/middleware/rate_limiter.py

from fastapi import HTTPException, Request
from redis import Redis
import time

class RateLimiter:
    """
    Rate limiting por usuario y tier.
    Free tier: 20 queries/hour
    Premium tier: 200 queries/hour
    """
    
    def __init__(self, redis_client: Redis):
        self.redis = redis_client
        self.limits = {
            "free": {"requests": 20, "window": 3600},
            "premium": {"requests": 200, "window": 3600},
        }
    
    async def check_limit(self, user_id: str, tier: str = "free"):
        """
        Verifica si el usuario ha excedido su límite.
        Usa sliding window algorithm.
        """
        limit_config = self.limits[tier]
        key = f"rate_limit:{tier}:{user_id}"
        
        now = int(time.time())
        window_start = now - limit_config["window"]
        
        # Remove old entries
        await self.redis.zremrangebyscore(key, 0, window_start)
        
        # Count requests in current window
        count = await self.redis.zcard(key)
        
        if count >= limit_config["requests"]:
            raise HTTPException(
                status_code=429,
                detail=f"Rate limit exceeded. Limit: {limit_config['requests']} requests per hour."
            )
        
        # Add current request
        await self.redis.zadd(key, {str(now): now})
        await self.redis.expire(key, limit_config["window"])
        
        return True

# Uso en endpoint
```

```python
class OptimizedQueriesRepository:
    """
    Queries optimizadas para casos comunes del copilot.
    Evita N+1 queries y carga solo datos necesarios.
    """
    
    async def get_dashboard_summary(self, time_range: int = 30) -> Dict:
        """
        Single query que obtiene todos los datos para dashboard summary.
        Evita múltiples round-trips a la DB.
        """
        async with self.db.acquire() as conn:
            result = await conn.fetchrow("""
                WITH 
                leads_stats AS (
                    SELECT 
                        COUNT(*) FILTER (WHERE status = 'pending_review') as pending,
                        COUNT(*) FILTER (WHERE status = 'approved') as approved,
                        COUNT(*) FILTER (WHERE status = 'rejected') as rejected,
                        AVG(ai_score) as avg_score
                    FROM leads
                    WHERE created_at >= NOW() - INTERVAL '%s days'
                ),
                analytics_stats AS (
                    SELECT 
                        COUNT(DISTINCT user_id) as visitors,
                        COUNT(*) as sessions,
                        AVG(session_duration) as avg_duration
                    FROM analytics_sessions
                    WHERE created_at >= NOW() - INTERVAL '%s days'
                ),
                content_stats AS (
                    SELECT 
                        COUNT(*) FILTER (WHERE status = 'pending_approval') as pending,
                        COUNT(*) FILTER (WHERE status = 'published') as published
                    FROM content
                    WHERE created_at >= NOW() - INTERVAL '%s days'
                )
                SELECT 
                    jsonb_build_object(
                        'leads', row_to_json(leads_stats.*),
                        'analytics', row_to_json(analytics_stats.*),
                        'content', row_to_json(content_stats.*)
                    ) as summary
                FROM leads_stats, analytics_stats, content_stats
            """, time_range, time_range, time_range)
            
            return result['summary']
```
